# Lawrie 项目国际化设计文档

## 概述

为 Lawrie 桌宠项目的 GitHub Pages 展示页面（index.html）添加中英双语支持，允许用户在页面上点击按钮切换语言，并将偏好保存到浏览器本地存储。

## 目标

- 支持中文（默认）和英文两种语言
- 提供直观的语言切换按钮
- 使用 localStorage 持久化保存用户语言偏好
- 保持现有单文件架构和像素风格
- 翻译风格保持简洁技术感

## 技术方案

### 架构选择

**方案**：纯 JavaScript 实现

**理由**：
- 零依赖，与当前单文件架构完美契合
- 代码量小（约 50-80 行增量代码）
- 性能好，无闪烁，即时切换
- 维护简单，所有翻译集中在一个字典对象中

### 核心机制

1. **翻译字典**：存储所有需要国际化的文本（中英双语）
2. **语言管理**：统一的 `setLanguage()` 函数更新所有文本元素
3. **持久化**：使用 localStorage 保存用户选择
4. **初始化**：页面加载时读取 localStorage，默认中文

## 功能实现

### 核心函数

#### 1. 语言设置函数

```javascript
function setLanguage(lang) {
  currentLang = lang;
  localStorage.setItem('language', lang);
  document.documentElement.lang = lang;
  
  // 更新静态文本
  document.title = i18n[lang].title;
  document.querySelector('header h1').textContent = i18n[lang].title;
  document.querySelector('header p').textContent = i18n[lang].subtitle;
  
  // 更新页脚（innerHTML 因为包含 <a> 和 <code>）
  const footer = document.querySelector('footer');
  footer.innerHTML = i18n[lang].footer1 + '<br>' + i18n[lang].footer2;
  
  // 更新按钮文本
  document.getElementById('langBtn').innerHTML = '🌐 ' + i18n[lang].langButton;
  
  // 重新渲染状态卡片
  renderCards();
}
```

#### 2. 卡片渲染函数

```javascript
function renderCards() {
  const grid = document.getElementById('grid');
  grid.innerHTML = ''; // 清空现有卡片
  
  const files = [
    'rabbit-idle.svg.html',
    'rabbit-typing.svg.html',
    'rabbit-thinking-v2.svg.html',
    'rabbit-sleeping.svg.html',
    'rabbit-happy.svg.html',
    'rabbit-error-v2.svg.html',
    'rabbit-notification.svg.html',
    'rabbit-carrying.svg.html'
  ];
  
  const states = i18n[currentLang].states;
  
  for (let i = 0; i < files.length; i++) {
    const card = document.createElement('div');
    card.className = 'card';
    
    const iframe = document.createElement('iframe');
    iframe.loading = 'lazy';
    iframe.title = states[i].name;
    iframe.addEventListener('load', () => fit(iframe));
    iframe.src = files[i];
    
    const meta = document.createElement('div');
    meta.className = 'meta';
    meta.innerHTML = `<div class="name">${states[i].name}</div><div class="desc">${states[i].desc}</div>`;
    
    card.appendChild(iframe);
    card.appendChild(meta);
    grid.appendChild(card);
    
    window.addEventListener('resize', () => fit(iframe));
  }
}
```

#### 3. 初始化逻辑

```javascript
// 页面加载时读取语言偏好
let currentLang = localStorage.getItem('language') || 'zh';
setLanguage(currentLang);

// 切换按钮点击事件
document.getElementById('langBtn').addEventListener('click', () => {
  setLanguage(currentLang === 'zh' ? 'en' : 'zh');
});
```

### 数据流

```
用户打开页面
    ↓
读取 localStorage['language'] || 默认 'zh'
    ↓
调用 setLanguage(lang)
    ↓
  ├─ 更新 <html lang>
  ├─ 更新静态文本（标题、副标题、页脚）
  ├─ 更新按钮文本
  └─ 调用 renderCards() 重新渲染状态卡片
    ↓
页面完全以目标语言显示
    ↓
用户点击切换按钮
    ↓
切换到另一语言，重复上述流程
```

## 技术细节

### HTML 结构变化

**原有结构**：
```html
<body>
  <header>...</header>
  <div class="grid" id="grid"></div>
  <footer>...</footer>
  <script>...</script>
</body>
```

**修改后**：
```html
<body>
  <button id="langBtn" class="lang-switch">🌐 English</button>
  <header>...</header>
  <div class="grid" id="grid"></div>
  <footer>...</footer>
  <script>...</script>
</body>
```

### 代码重构点

1. **状态数组重构**
   - 原代码：硬编码的中文数组 `states`
   - 新代码：从 `i18n[currentLang].states` 动态读取

2. **卡片渲染逻辑**
   - 原代码：`for...of` 遍历 states 数组
   - 新代码：封装为 `renderCards()` 函数，支持重新渲染

3. **文本更新**
   - 静态文本：通过 `textContent` 或 `innerHTML` 更新
   - 动态内容：通过重新渲染卡片实现

## 兼容性

- **浏览器支持**：所有现代浏览器（Chrome、Firefox、Safari、Edge）
- **localStorage**：IE8+ 支持
- **降级方案**：localStorage 不可用时，语言切换仍可用，但不持久化

## 性能考虑

- **初始加载**：增加约 1-2KB 的翻译字典
- **切换性能**：纯 DOM 操作，即时完成，无闪烁
- **内存占用**：翻译字典常驻内存，占用可忽略（<10KB）

## 维护性

- **翻译集中管理**：所有翻译在 `i18n` 对象中，修改时中英文对照清晰
- **扩展性**：未来添加新语言只需在 `i18n` 中添加新的语言键
- **代码复用**：`setLanguage()` 和 `renderCards()` 可复用到其他页面

## 风险与限制

**风险**：
- 翻译质量依赖人工审核
- 用户清除浏览器数据会丢失语言偏好

**限制**：
- 仅支持中英双语（未来可扩展）
- 不支持 SEO 优化（仍是单页面单 URL）
- 不支持根据浏览器语言自动判断（首次访问默认中文）

**缓解措施**：
- 翻译由人工审核，保持像素风技术感
- localStorage 丢失后回退到默认中文，不影响使用
- 未来可通过 `navigator.language` 实现自动语言检测

## 总结

该设计通过纯 JavaScript 实现中英双语切换，保持了项目的单文件架构和像素风格。实现简洁高效，用户体验流畅，维护成本低。所有翻译保持简洁技术感，符合项目整体风格。
